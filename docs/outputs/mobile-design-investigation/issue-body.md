## 概要

デザインシステム（`knowledge/design-system.md`）とデザインディレクション（`knowledge/design-direction.md`）では**モバイルファースト**を設計原則として定義しているが、実際のテンプレートとコマンドはすべて**デスクトップ固定幅（1280px）**のみに対応しており、設計思想と実装が矛盾している。

このイシューでは、モバイル・レスポンシブ対応に必要な変更箇所を調査・整理し、実装計画を策定する。

### 現状の矛盾

| 定義（あるべき姿） | 実装（現状） |
|---|---|
| `design-system.md`: ブレークポイント定義済み（sm:640px, md:768px, lg:1024px, xl:1280px, 2xl:1536px） | テンプレート・コマンドで一切使用されていない |
| `design-system.md`: 「モバイルファースト: デフォルトスタイルはモバイル。ブレークポイントで拡張する」 | テンプレートは `width: 1280px` 固定 |
| `design-direction.md`: 「Responsive First — モバイルから設計し、デスクトップに拡張する」 | コマンドに「レスポンシブ対応不要」と明記 |
| `design-direction.md`: 横スクロール・小フォントをアンチパターンと指定 | 固定幅テンプレートではモバイル表示を検証できない |

### 影響範囲

### 設計フェーズ

- [ ] テンプレートのレスポンシブ化方針を決定する
  - 方針A: 1つのレスポンシブ HTML テンプレート（メディアクエリで切替）
  - 方針B: デバイスごとに別テンプレート（mobile.html / tablet.html / desktop.html）
  - 方針C: デスクトップ固定を維持しつつ、オプションでモバイル版を追加生成
- [ ] コマンドのデバイス選択 UI を設計する

- [ ] 案件種別による対応要否の判断フローを設計する

- [ ] `design-system.md` のブレークポイント定義をテンプレートに反映する方法を設計する

## 設計上の考慮事項

1. **段階的導入**: 既存のデスクトップ固定ワークフローを壊さず、オプションとしてモバイル対応を追加する方向が安全
2. **案件種別による要否判断**: すべての案件にモバイル対応が必要なわけではない。社内システム・管理画面はデスクトップのみで十分なケースが多く、LP・商用サイト・アプリ開発ではモバイル対応が必須。コマンド実行時に案件の性質に応じた選択肢を提示する
3. **diagram.html は対象外の可能性**: ダイアグラムはデバイス非依存のため、レスポンシブ化の優先度は低い
4. **generate-screens との連携**: 画面一覧生成時にデバイスごとのレイアウト差異（サイドバーの折りたたみ、ハンバーガーメニュー等）を考慮する必要がある
5. **デザインシステムとの整合性**: `design-system.md` で定義済みのブレークポイント・サイドバー幅（固定 240px、折りたたみ時 64px）をそのまま活用する
