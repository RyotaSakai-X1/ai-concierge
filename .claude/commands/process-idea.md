# /process-idea — アイデア精査・仕様書生成

## 処理フロー

1. `docs/ideas.md` を読み込む
2. `## ステータス: 未処理` のアイデアをすべて抽出する
3. 各アイデアに対して以下を実行する：

### 評価軸（スコアリング）

| 軸 | 評価内容 |
|---|---|
| インパクト | 実現したときのビジネス価値（高/中/低） |
| 工数 | 実装にかかる概算時間（S/M/L/XL） |
| リスク | 技術的・ビジネス的な不確実性 |
| 依存関係 | 他のタスク・システムへの依存 |

### 出力ファイル

`docs/specs/YYYY-MM-DD-{アイデア名をスラッグ化}.md` を以下のフォーマットで生成する。
フォーマットは `knowledge/templates/spec.md` を参照。

### 後処理

- 処理済みアイデアの `## ステータス:` を `未処理` → `仕様化済み` に更新する
- `docs/roadmap.md` の適切な優先度の位置に1行追記する

## 引数

- 引数なし: 未処理アイデアをすべて処理
- ファイルパス指定: そのファイルのアイデアのみ処理
- `--dry-run`: ファイル生成せず評価結果だけ表示
