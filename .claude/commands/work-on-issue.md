# /work-on-issue — イシュー起点の実装〜PR 作成

イシュー番号を受け取り、分析 → 実装 → セルフレビュー → PR 作成を一気通貫で実行する。スタンドアロン実行と `/parallel-work` からのチームメンバー実行に対応。

## 引数

- `$ARGUMENTS` — イシュー番号（例: `12`）

## 動作モード

| モード | 判定条件 | ブランチ作成 | プッシュ・PR 作成 |
|--------|---------|------------|-----------------|
| 通常実行 | worktree でない かつ main ブランチ | する | する |
| 既存ブランチ上 | worktree でない かつ main 以外 | スキップ（現ブランチで続行） | する |
| チームメンバー実行 | worktree 内（`/parallel-work` 経由） | スキップ | スキップ（テックリードが行う） |

判定方法: `git rev-parse --git-dir` と `git rev-parse --git-common-dir` が異なれば worktree 内。

## Step 1: イシュー分析

入力: `gh issue view $ARGUMENTS --json title,body,labels,assignees,milestone`

抽出項目:

| 項目 | 内容 |
|------|------|
| ゴール | 何を達成するか |
| やること | 具体的なタスク一覧（チェックリスト） |
| 影響範囲 | どのファイル・機能に影響するか |
| 依存関係 | 前提となるイシューや機能 |

不明点は仮定を置いて進める（`> ⚠️ 仮定：〜` で明示）。

## Step 2: ブランチ作成（通常実行時のみ）

ブランチ名: `feature/YYYY-MM-DD-{slug}`（`.claude/rules/git-workflow.md` の命名規則に従う）

| アクション | コマンド |
|-----------|---------|
| ブランチ作成 | `git checkout -b feature/YYYY-MM-DD-{slug}` |
| イシューアサイン | `gh issue edit $ARGUMENTS --add-assignee @me` |
| ステータス変更 | `.claude/rules/git-workflow.md` の手順で **In progress** に変更 |

## Step 3: 実装

チームメンバー実行時は実装開始前に変更予定ファイル一覧を共有タスクリストに宣言する。

イシューの「やること」に従い順次実装する。

中断条件: エラー発生 / 想定外の状態 / 判断に必要な情報の不足 → 自力リカバリせず処理を中断しテックリードに報告。

## Step 4: セルフレビュー

`/reviewing` を使い、変更内容をレビューする。

| レビュー対象 | 入力 |
|------------|------|
| 変更差分 | `git diff`（未コミットの場合） |

`/reviewing` の判定結果に応じた対応:

| 判定 | 対応 |
|------|------|
| ✅ 承認 | Step 5 へ進む |
| ⚠️ 条件付き承認 | 指摘箇所を修正し、再度 `/reviewing` |
| ❌ 差し戻し | 指摘箇所を修正し、再度 `/reviewing` |

## Step 5: コミット・PR 作成

| モード | 処理 |
|--------|------|
| チームメンバー実行 | `git add -A` → `git commit` → コミットハッシュと変更ファイル一覧を共有タスクリストに報告して終了 |
| 通常実行・既存ブランチ | `/creating-pr` に委譲（PR 本文に `Closes #$ARGUMENTS` + セルフレビューサマリを含める） |

## Step 6: 結果レポート

テンプレート: `knowledge/templates/work-on-issue-report.md`

| モード | 報告先 |
|--------|--------|
| 通常実行 | チャットに表示 |
| チームメンバー実行 | 共有タスクリスト経由（コミットハッシュ、変更ファイル一覧を含む） |

## 制約

- force push・ブランチ削除は行わない
- `.env` や秘密情報を含むファイルがあれば警告して除外する
- 実装が大きすぎる場合は最小限の変更に留め、残りを別イシューとして提案する
- コミットメッセージは日本語で記述する
