# /executing-plan — タスクリストを順次自律実行

引数のタスクファイル（`docs/outputs/*/tasks.md`）を読み込み、未完了タスク `[ ]` を順番に実行する。

## 引数

- `$ARGUMENTS` — タスクファイルのパス（例: `docs/outputs/feedback-system/tasks.md`）

## 動作モード

| モード | 引数 | 動作 |
|--------|------|------|
| 通常実行 | パスのみ | 未完了タスクを全件順次実行 |
| ドライラン | `--dry-run` 付き | 実行計画のみ表示（ファイル変更なし） |
| 単体実行 | `--task N` 付き | 指定番号のタスクのみ実行 |

## Step 1: タスクファイル読み込み

入力: `$ARGUMENTS` で指定されたタスクファイル

抽出項目:
- 全タスク一覧（番号、タイトル、完了状態）
- 未完了タスク `[ ]` の一覧

ドライラン時はここで実行計画を表示して終了。

## Step 2: タスク実行

タスク種別と実行方法:

| タスク種別 | 実行内容 | 出力先 |
|-----------|---------|--------|
| コード生成 | 仕様に従いファイルを作成 | `docs/outputs/{slug}/` 配下 |
| 設計書 | テンプレートに従い Markdown を生成 | `docs/outputs/{slug}/` 配下 |
| 調査 | Web 検索し結果を構造化 | `docs/outputs/{slug}/research.md` |

各タスク完了後、`tasks.md` の `[ ]` を `[x]` に更新する。

### 事前確認が必要な操作

以下の操作は AskUserQuestion で確認してから実行する。

| 操作 | 選択肢 |
|------|--------|
| 外部 API へのリクエスト | `実行する` / `スキップ` |
| データの削除・上書き | `実行する` / `スキップ` |
| 本番環境への変更 | `実行する` / `中止` |

### エラー発生時の対応

| エラー種別 | 対応 |
|-----------|------|
| タスクファイルが存在しない | エラーメッセージを表示して終了 |
| タスクの実行に失敗 | 失敗したタスクを `[ ]` のまま残し、エラー内容をレポートに記録して次のタスクへ進む |
| 全タスク完了済み（未完了なし） | 「未完了タスクはありません」と表示して終了 |
| 依存タスクが未完了 | 依存先を先に実行するか AskUserQuestion で確認（`依存先を先に実行` / `このタスクをスキップ`） |

## Step 3: 実行レポート

出力先: `docs/outputs/{slug}/execution-report.md`

レポート内容:

| 項目 | 内容 |
|------|------|
| 実行日時 | ISO 8601 形式 |
| 完了タスク数 / 全タスク数 | 例: 3/5 |
| 生成したファイル一覧 | パスの箇条書き |
| 失敗したタスク | タスク番号とエラー内容 |
| 未解決の問題・次のアクション | 残タスクや対応が必要な事項 |

## 制約

- タスクファイルのパスが `docs/outputs/` 配下でない場合は実行しない
- 各タスクの実行結果は必ず `docs/outputs/{slug}/` 配下に出力する
- タスク実行中に仕様の曖昧さを発見した場合、仮定を置いて進め `> ⚠️ 仮定：〜` で明示する
- force push・ブランチ削除は行わない
