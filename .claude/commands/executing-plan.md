# /executing-plan — タスクリストを順次自律実行

引数のタスクファイル（`docs/outputs/*/tasks.md`）を読み込み、未完了タスク `[ ]` を順番に実行する。

## 引数

- `$ARGUMENTS` — タスクファイルのパス（例: `docs/outputs/feedback-system/tasks.md`）
  - `--dry-run`: 実行せず実行計画のみ表示
  - `--task N`: タスク番号を指定して単体実行

## Step 1: タスク実行

入力: `$ARGUMENTS` で指定されたタスクファイル

タスク種別と実行方法:

| タスク種別 | 実行内容 |
|-----------|---------|
| コード生成 | ファイルを作成し `docs/outputs/{slug}/` に配置 |
| 設計書 | Markdown ファイルを生成 |
| 調査 | Web 検索し `research.md` にまとめる |

各タスク完了後、`tasks.md` の `[ ]` を `[x]` に更新する。

事前確認が必要な操作:

| 操作 |
|------|
| 外部 API へのリクエスト |
| データの削除・上書き |
| 本番環境への変更 |

## Step 2: 実行レポート

出力先: `docs/outputs/{slug}/execution-report.md`

レポート内容:

| 項目 |
|------|
| 実行日時 |
| 完了タスク数 / 全タスク数 |
| 生成したファイル一覧 |
| 未解決の問題・次のアクション |
