# /executing-plan — タスクリストを順次自律実行

## 処理フロー

引数として渡されたタスクファイル（`docs/outputs/*/tasks.md`）を読み込み、
未完了タスク `[ ]` を順番に実行する。

### 実行ルール

- コード生成タスク → 実際にファイルを作成して `docs/outputs/{slug}/` に配置する
- 設計書タスク → Markdown ファイルを生成する
- 調査タスク → Web 検索して結果を `research.md` にまとめる
- 各タスク完了後 → `tasks.md` の `[ ]` を `[x]` に更新する
- **以下のタスクは必ず事前確認する：**
  - 外部APIへのリクエスト
  - データの削除・上書き
  - 本番環境への変更

### 実行後レポート

完了したら `docs/outputs/{slug}/execution-report.md` に以下を記録する：

- 実行日時
- 完了タスク数 / 全タスク数
- 生成したファイル一覧
- 未解決の問題・次のアクション

## 使用例

`/executing-plan docs/outputs/feedback-system/tasks.md` で実行する。

## 引数オプション

- `--dry-run`: 実行せず、実行計画だけ表示する
- `--task 3`: タスク番号を指定して単体実行する
