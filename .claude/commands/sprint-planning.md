# スプリント計画コマンド

着手可能なイシューから、直近の作業計画（スプリント）を作成します。

## 使用方法

```bash
/sprint-planning
/sprint-planning --days 5
/sprint-planning --max 3
```

オプション:
- `--days N`: 計画期間（デフォルト: 5日 = 1週間）
- `--max N`: 最大イシュー数（デフォルト: 5件）

## 引数の形式: $ARGUMENTS

## 実行手順

### Step 1: 着手可能なイシューを取得

`/triaging` と同じロジックで、全オープンイシューから着手可能（🟢）なものを抽出する。

### Step 2: 工数の推定

各イシューの「やること」チェックリストの項目数から工数を推定する:

| チェック項目数 | 推定工数 |
|-------------|---------|
| 1〜2 | S（半日〜1日） |
| 3〜4 | M（1〜2日） |
| 5〜7 | L（3〜5日） |
| 8以上 | XL（1週間以上） |

### Step 3: スプリントに入れるイシューを選定

以下の基準で選定する:

1. `/triaging` の優先度順に並べる
2. 推定工数の合計が `--days` に収まる範囲で詰める
3. `--max` を超えない

### Step 4: スプリント計画を提示

```markdown
## スプリント計画（YYYY-MM-DD 〜 YYYY-MM-DD）

### 今回取り組むイシュー

| 優先 | # | タイトル | 推定工数 | 着手順の理由 |
|------|---|---------|---------|------------|
| 1 | #XX | ... | S | ブロッカー解消のため最優先 |
| 2 | #XX | ... | M | ... |
| 3 | #XX | ... | S | ... |

**合計推定工数: X日分**

### 今回見送るイシュー

| # | タイトル | 理由 |
|---|---------|------|
| #XX | ... | 工数オーバー |
| #XX | ... | Phase 2 以降 |

### 依存関係マップ

{着手順に依存関係がある場合、順序を図示}
```

### Step 5: 次のアクション

スプリント計画を承認するか確認し、承認されたら:

1. 最優先イシューの `/working-on-issue` or ブランチ作成手順を案内
2. 必要に応じてスプリント計画を `docs/outputs/sprint/` に保存

## 注意事項

- 工数は推定であり、実績と乖離する可能性がある
- スプリント計画はあくまで提案。ユーザーが自由に入れ替えてよい
- 1人で作業する前提の計画。複数人の場合はユーザーが調整する
