# イシュー優先度整理コマンド

全オープンイシューを優先度・依存関係で分析し、着手順を提案します。

## 使用方法

```bash
/triaging
```

## 引数の形式: $ARGUMENTS

## 実行手順

### Step 1: 全オープンイシューを取得

`gh issue list --repo RyotaSakai-X1/ai-concierge --state open --json number,title,body,labels` で全オープンイシューを取得する。

### Step 2: 依存関係の解析

各イシューの本文から以下を抽出する:

- **「関連」セクション** のイシュー番号（`#XX`）
- **「前提条件」セクション** のイシュー番号
- **本文中の** `#XX` 参照

依存関係をグラフとして整理する。

### Step 3: 各イシューの状態判定

| 状態 | 条件 |
|------|------|
| 🟢 着手可能 | 依存イシューがすべてクローズ済み、または依存なし |
| 🟡 依存解決待ち | 依存イシューにオープンのものがある |
| 🔵 Phase 2 以降 | タイトルに【Phase 2】【Phase 3】を含む |

### Step 4: 優先度のソート

着手可能なイシューを以下の基準で並び替える（上が高優先）:

1. **ブロッカー度**: 他のイシューから多く依存されているものが最優先
2. **Phase**: Phase 1 > Phase 2 > Phase 3
3. **工数の小ささ**: 小さいタスクを先に片付ける（やること欄のチェック数で推定）

### Step 5: 結果を提示

以下の形式で結果を提示する:

```markdown
## イシュー優先度整理結果

### 🟢 今すぐ着手可能（優先度順）

| 優先 | # | タイトル | 依存 | ブロッカー度 | 推奨理由 |
|------|---|---------|------|------------|---------|
| 1 | #XX | ... | なし | 高（N件が依存） | ... |
| 2 | #XX | ... | なし | 中 | ... |

### 🟡 依存解決待ち

| # | タイトル | 待っているイシュー |
|---|---------|-----------------|
| #XX | ... | #YY, #ZZ |

### 🔵 Phase 2 以降（今は対象外）

| # | タイトル | Phase |
|---|---------|-------|
| #XX | ... | 2 |
```

### Step 6: 次のアクションの提案

最優先イシューについて、以下を提案する:

- `/working-on-issue {番号}` が使える場合はそのコマンドを案内
- 使えない場合はブランチ作成の手順を案内
- そのイシューの「やること」チェックリストを表示

## 注意事項

- 優先度はあくまで提案であり、最終判断はユーザーが行う
- `docs/roadmap.md` の Phase 分類と矛盾しないようにする
