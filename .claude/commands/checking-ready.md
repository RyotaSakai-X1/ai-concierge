# イシュー着手可否チェックコマンド

指定したイシューが着手可能な状態かどうかを判定します。

## 使用方法

```bash
/checking-ready <イシュー番号>
```

例:

- `/checking-ready 12`
- `/checking-ready 16`

## 引数の形式: $ARGUMENTS

## 実行手順

### Step 1: イシュー情報の取得

`gh issue view {番号} --repo RyotaSakai-X1/team-design-test --json number,title,body,state` で対象イシューの情報を取得する。

### Step 2: 依存イシューの特定

本文から依存しているイシュー番号（`#XX`）を抽出する。
抽出元:
- 「関連」セクション
- 「前提条件」セクション
- 本文中の参照

### Step 3: 依存イシューの状態確認

各依存イシューについて `gh issue view` で状態（open/closed）を確認する。

### Step 4: 着手可否の判定

| 判定 | 条件 |
|------|------|
| ✅ 着手可能 | 依存イシューがすべてクローズ済み、または依存なし |
| ⚠️ 一部未完了 | 依存の一部がオープンだが、並行着手できる可能性あり |
| ❌ 着手不可 | 必須の前提イシューがオープン |

### Step 5: 結果を提示

```markdown
## #XX 着手可否チェック

**判定: ✅ 着手可能 / ⚠️ 一部未完了 / ❌ 着手不可**

### 依存イシューの状態

| # | タイトル | 状態 |
|---|---------|------|
| #YY | ... | ✅ クローズ済み |
| #ZZ | ... | ❌ オープン |

### やること（チェックリスト）

- [ ] タスク1
- [ ] タスク2
...

### 次のアクション

{判定に応じた案内}
```

## 注意事項

- 依存関係の判定はイシュー本文ベースなので、本文に記載がない暗黙の依存は検出できない
- 判定結果はあくまで参考。ユーザーの判断で着手してよい
