# /checking-ready — イシュー着手可否チェック

指定イシューが着手可能な状態かどうかを判定し、依存関係と合わせて結果を表示する。

## 引数

- `$ARGUMENTS` — イシュー番号（例: `12`）

## Step 1: イシュー情報と依存関係の取得

入力: `gh issue view {番号} --repo RyotaSakai-X1/Claude-Factory --json number,title,body,state`

依存イシューの抽出元:
- 「関連」セクション
- 「前提条件」セクション
- 本文中の `#XX` 参照

各依存イシューの状態を `gh issue view` で確認する。

## Step 2: 着手可否の判定

| 判定 | 条件 |
|------|------|
| ✅ 着手可能 | 依存イシューがすべてクローズ済み、または依存なし |
| ⚠️ 一部未完了 | 依存の一部がオープンだが、並行着手できる可能性あり |
| ❌ 着手不可 | 必須の前提イシューがオープン |

## Step 3: 結果の表示

出力構成:

| セクション | 内容 |
|-----------|------|
| `## #XX 着手可否チェック` | タイトル |
| 判定 | ✅ / ⚠️ / ❌ |
| 依存イシューの状態 | テーブル: `#` / `タイトル` / `状態`（✅ クローズ済み / ❌ オープン） |
| やること | イシュー本文から抽出したチェックリスト |
| 次のアクション | 判定に応じた案内 |

## 制約

- 依存関係はイシュー本文ベース。本文に記載がない暗黙の依存は検出できない
- 判定結果はあくまで参考。ユーザーの判断で着手してよい
