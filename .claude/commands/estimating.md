# 工数見積コマンド

機能要件が記載されたスプレッドシートに対して工数見積を人日単位で行います。

## 使用方法

```bash
/estimating <スプレッドシートのURL または ID>
```

例:

- `/estimating https://docs.google.com/spreadsheets/d/1ABC123xyz/edit?gid=0#gid=0`
- `/estimating 1ABC123xyz`

## 引数の形式: $ARGUMENTS

## 実行手順

### Step 1: スプレッドシート ID の抽出

引数からスプレッドシート ID を抽出してください:

- **URL の場合**: `/spreadsheets/d/` と `/edit` の間の文字列が ID
  - 例: `https://docs.google.com/spreadsheets/d/1ABC123xyz/edit` → `1ABC123xyz`
- **ID の場合**: そのまま使用

また、gid パラメータがあれば対象シートの特定に使用してください。

### Step 2: スプレッドシートの内容を取得

`mcp__google-sheets__list_sheets` でシート一覧を取得し、次に `mcp__google-sheets__get_sheet_data` で機能要件が記載されているシートのデータを取得してください。

```
spreadsheet_id: <抽出したID>
sheet: <対象シート名>
```

**注意**: シートに複数のタブがある場合は、機能要件や見積りに関連するシートを優先して取得してください。

### Step 3: 機能要件の分析

取得したデータから以下を分析してください:

1. **機能一覧の特定**: 各行が 1 つの機能を表しているか確認
2. **カテゴリ/グループの把握**: 機能がカテゴリ別に分類されているか
3. **既存の列構造**: すでに見積り列がある場合はその形式を確認
4. **要件の詳細度**: 各機能の記載内容から複雑度を判断

### Step 4: 工数見積の基準

以下の基準で各機能の工数を見積もってください:

#### 機能タイプ別の基準工数

**基本的に機能要件は API ＋ UI です**

| タイプ       | 内容                                                 | 基準工数   |
| ------------ | ---------------------------------------------------- | ---------- |
| 標準的な UI  | CRUD 画面、一覧/詳細画面                             | 1〜2 人日  |
| 複雑な UI    | ダッシュボード、複雑なフォーム、インタラクティブ機能 | 3〜5 人日  |
| 標準的な API | 複数テーブル結合、バリデーション                     | 2〜3 人日  |
| 複雑な API   | 複雑なビジネスロジック、外部連携                     | 4〜5 人日  |
| バッチ処理   | 定期実行、データ処理                                 | 2〜5 人日  |
| 認証/認可    | ログイン、権限管理                                   | 2〜5 人日  |
| 外部連携     | API 連携、ファイル取込                               | 2〜10 人日 |

#### 複雑度の調整係数

- **シンプル** (明確な要件、既存パターンあり): ×0.8
- **標準** (一般的な要件): ×1.0
- **複雑** (曖昧な要件、新規技術): ×1.5
- **非常に複雑** (未知の領域、高リスク): ×2.0

### Step 6: スプレッドシートへの工数列書き出し（必須）

見積り確定後、**必ず**スプレッドシートに工数を書き出してください。

#### 6-1: 書き出し先の列を特定

スプレッドシートのヘッダー行を確認し、以下の列を特定:

| 列名パターン                              | 用途             |
| ----------------------------------------- | ---------------- |
| `工数` / `見積` / `人日` / `工数（人日）` | 工数を書き出す列 |

**列が存在しない場合**:

1. `mcp__google-sheets__add_columns` で最終列の右に 1 列追加
2. ヘッダー行に「工数（人日）」を設定

```
spreadsheet_id: <ID>
sheet: <シート名>
count: 1
start_column: <最終列のインデックス + 1>
```

#### 6-2: 工数データの書き出し

`mcp__google-sheets__update_cells` を使用して工数を書き出し:

```
spreadsheet_id: <ID>
sheet: <シート名>
range: <工数列の範囲>  例: "F2:F100"
data: [
  [1.0],   // 行2: 工数
  [2.0],   // 行3
  [0.5],   // 行4
  ...
]
```

**重要**:

- データ行のみ書き出し（ヘッダー行は除く）
- 機能要件の行と工数の行が正しく対応していることを確認
- 空行や合計行がある場合はスキップまたは適切に処理

#### 6-3: 書き出し完了の確認

書き出し後、以下を報告してください:

```
## スプレッドシート書き出し完了

- 書き出し先: [シート名] の [列範囲]
- 書き出し件数: XX件
- 工数合計: XX.X人日
```

#### 6-4: エラー時の対応

- **権限エラー**: スプレッドシートの編集権限を確認するよう依頼
- **範囲エラー**: 行数・列数を再確認し、正しい範囲で再実行
- **データ型エラー**: 数値が文字列になっていないか確認

## 計算ルール

- **バッファ**: 素の工数に対して **+20%** を加算
  - 例: 1.0 人日 → 1.2 人日（バッファ込み）
- **端数処理**: 小数点第 1 位まで表示（第 2 位を四捨五入）
- **最小単位**: 0.5 人日（半日）

## 注意事項

1. **要件が曖昧な場合**: 見積り時に前提条件を明記し、幅を持たせる（例: 1〜2 人日）
2. **依存関係がある場合**: 先行機能の完了を前提とした見積りであることを備考に記載
3. **技術調査が必要な場合**: 調査工数を別途計上することを提案
4. **テスト工数**: 上記基準には単体テストを含む。結合テスト・システムテストは別途見積りが必要か確認

## エラーハンドリング

- **スプレッドシートにアクセスできない場合**: 権限設定の確認を依頼
- **機能要件の形式が不明な場合**: シートの構造についてユーザーに確認
- **見積り対象が多すぎる場合**: カテゴリ別に分割して段階的に見積り
