# レビューコマンド

仕様書や成果物をチェックリストに基づいてレビューし、問題点や改善案を提示します。

## 使用方法

```bash
/reviewing <対象ファイルのパス>
```

例:

- `/reviewing docs/specs/2026-03-09-hello-world-automation.md`
- `/reviewing docs/outputs/hello-world-automation/execution-report.md`

## 引数の形式: $ARGUMENTS

## 実行手順

### Step 1: 対象の判定

引数からレビュー種別を自動判定する:

| 引数の形式 | レビュー種別 | 使用するチェックリスト |
|-----------|------------|-------------------|
| `docs/specs/...` | 仕様書レビュー | `knowledge/review/spec-checklist.md` |
| `docs/outputs/...` | 成果物レビュー | `knowledge/review/output-checklist.md` |
| `#XX` または PR番号 | PRコードレビュー | 下記の PR レビュー手順 |

判定できない場合は、ユーザーにどのレビューか確認する。

#### PRコードレビューの場合

PR番号が指定された場合は、以下の手順で PRコードレビューを行う:

1. `gh pr view {番号} --json title,body,files,additions,deletions,commits` でPR情報を取得
2. `gh pr diff {番号}` で差分を取得
3. 以下の観点でレビュー:
   - **要件充足**: PR本文に記載された目的を達成しているか
   - **コード品質**: 命名、構造、重複、可読性
   - **セキュリティ**: 秘密情報、インジェクション、OWASP Top 10
   - **テスト**: テストの有無と十分性
   - **影響範囲**: 意図しない副作用がないか
4. レビュー結果をチャットに表示
5. `gh pr review {番号} --comment --body "レビュー結果"` でPRにコメント

以降の Step 2〜6 は仕様書/成果物レビューの場合に適用する。

### Step 2: 関連ファイルの読み込み

以下のファイルを読み込む:

1. **対象ファイル**（引数で指定されたもの）
2. **チェックリスト**（Step 1 で判定したもの）
3. **ビジネスルール**（`knowledge/business-rules.md`）
4. **仕様書**（成果物レビューの場合、対応する仕様書も読む）

成果物レビューの場合、対応する仕様書の特定方法:
- `docs/outputs/{slug}/` の `{slug}` と一致する `docs/specs/*-{slug}.md` を探す

### Step 3: チェックリストに基づくレビュー

チェックリストの各項目について、対象ファイルの内容を評価する。

各項目に対して以下を判定:
- **✅**: 問題なし
- **⚠️**: 軽微な問題あり（改善推奨）
- **❌**: 重大な問題あり（修正必須）

### Step 4: 総合判定

すべてのチェック結果から総合判定を下す:

| 条件 | 総合判定 |
|------|---------|
| ❌ が 0 件、⚠️ が 2 件以下 | ✅ 承認 |
| ❌ が 0 件、⚠️ が 3 件以上 | ⚠️ 条件付き承認 |
| ❌ が 1 件以上 | ❌ 差し戻し |

### Step 5: レビュー結果の出力

1. `knowledge/templates/review-report.md` のテンプレートに従ってレビュー結果を作成
2. 結果をチャットに表示する
3. 対象ファイルと同じディレクトリに `review-report.md` として保存する
   - 仕様書の場合: `docs/specs/review-{元ファイル名}.md`
   - 成果物の場合: `docs/outputs/{slug}/review-report.md`

### Step 6: 次のアクションの提示

判定結果に応じて、次のステップを案内する:

| 判定 | 案内する次のアクション |
|------|-------------------|
| ✅ 承認 | 仕様書 → 「`/executing-plan` で実行に進めます」 / 成果物 → 「`/creating-pr` で PR を作成できます」 |
| ⚠️ 条件付き承認 | 指摘箇所の修正案を提示し、修正後に再レビューを提案 |
| ❌ 差し戻し | 問題の原因と修正の方向性を提示 |

## 注意事項

- レビューはあくまで AI による自動チェックであり、最終判断はユーザーが行う
- ❌ 差し戻しの場合でも、ユーザーが「このまま進める」と判断すれば従う
- チェックリストにない観点でも、明らかな問題を発見した場合は指摘する
