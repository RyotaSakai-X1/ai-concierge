# /create-estimate-doc — 工数見積もりから見積書を自動生成

`/estimating` で算出した工数データ（スプレッドシート）と案件情報をもとに見積書を生成する。

## 引数

- `$ARGUMENTS` — `/estimating` 済みスプレッドシート URL / ID（省略時は対話形式で収集）

## Step 1: 工数データの取得

入力: `$ARGUMENTS` からスプレッドシート ID を抽出（URL の場合は `/spreadsheets/d/{ID}/edit` の `{ID}` 部分）

取得手順: `mcp__google-workspace__get_spreadsheet_info` → `mcp__google-workspace__read_sheet_values`

抽出項目:

| 項目 | 抽出元 |
|------|--------|
| 機能名 | 機能要件の列 |
| カテゴリ | カテゴリ/グループ列 |
| 工数（人日） | 工数列（`/estimating` が書き出した値） |
| 備考 | 備考列（あれば） |

引数なしの場合は AskUserQuestion でスプレッドシート URL を入力してもらう。

## Step 2: 案件情報の収集

AskUserQuestion で以下を収集:

| 項目 | 形式 |
|------|------|
| クライアント名 | テキスト入力 |
| プロジェクト名 | テキスト入力 |
| 作成者名 | テキスト入力 |
| 有効期限 | デフォルト: 見積日より 30 日間 |

単価の選択肢:

| 選択肢 | 用途 |
|--------|------|
| ¥50,000 /人日 | ジュニアエンジニア |
| ¥70,000 /人日 | ミドルエンジニア |
| ¥100,000 /人日 | シニアエンジニア |
| カスタム金額を入力 | 上記以外 |

フェーズ分割:

| 選択肢 | 説明 |
|--------|------|
| フェーズ分割なし | 全機能を 1 期で見積もり |
| フェーズ分割あり | カテゴリや優先度でフェーズに分割 |

## Step 3: 金額の計算

| 項目 | 計算方法 |
|------|---------|
| 各機能の金額 | 工数 × 単価 |
| 小計（税抜） | 全機能の金額の合計 |
| 消費税 | 小計 × 10% |
| 合計（税込） | 小計 + 消費税 |

端数処理: 各機能の金額・消費税ともに 1 円未満切り捨て。

フェーズ分割ありの場合: カテゴリ/優先度でグルーピングし、各フェーズの工数・金額を集計。1 人日 = 1 営業日として期間を概算。

## Step 4: 見積書の生成

テンプレート: `knowledge/templates/estimate-doc.md`

自動的に含める前提条件:
- 要件変更時は再見積り
- テスト工数は単体テストを含む（結合/システムテストは別途）
- インフラ・サーバー費用は含まない
- デザイン制作費は含まない

備考欄に「スコープ外」「別途」等の記載がある項目は「含まれないもの」に自動抽出する。

出力先: `docs/outputs/{案件slug}/estimate-doc.md`

## Step 5: 結果の報告

報告項目: 出力ファイルパス / 総工数 / 単価 / 合計金額（税込） / 機能数 / フェーズ数

次のアクション:

| アクション | コマンド |
|-----------|---------|
| 見積書のレビュー | `/reviewing docs/outputs/{案件slug}/estimate-doc.md` |
| PR 作成 | `/creating-pr 見積書を作成` |

## 制約

- 単価は必ずユーザーに確認する
- 工数データが空欄の機能がある場合は警告し、先に `/estimating` の実行を提案
- 金額の端数処理は一貫して切り捨て
- 見積書は概算である旨を備考に記載

## 関連コマンド

- `/estimating` — 工数見積もりを行う（この入力のスプレッドシートを生成）
- `/define-requirements` — 要件定義書を生成する（パイプライン上流）
- `/reviewing` — 見積書のレビュー
- `/creating-pr` — 見積書を PR として提出
