# /create-estimate-doc — 工数見積もりから見積書を自動生成

`/estimating` で算出した工数データ（スプレッドシート）と案件情報をもとに、見積書を自動生成します。

## 引数

- `$ARGUMENTS` — 以下のいずれか:
  - `/estimating` 済みのスプレッドシート URL
  - スプレッドシート ID
  - 引数なし（対話形式で情報を収集）

## 実行手順

### Step 1: 工数データの取得

#### スプレッドシートが指定された場合

1. 引数からスプレッドシート ID を抽出する（URL の場合は `/spreadsheets/d/` と `/edit` の間）
2. `mcp__google-workspace__get_spreadsheet_info` でシート構造を確認
3. `mcp__google-workspace__read_sheet_values` で機能要件と工数データを取得
4. 以下の情報を抽出する:

| 抽出対象 | 抽出元 |
|---------|--------|
| 機能名 | 機能要件の列 |
| カテゴリ | カテゴリ/グループ列 |
| 工数（人日） | 工数列（`/estimating` が書き出した値） |
| 備考 | 備考列（あれば） |

#### 引数なしの場合

AskUserQuestion でスプレッドシートの URL を入力してもらう。

### Step 2: 案件情報の収集

AskUserQuestion で以下の情報を収集する:

1. **クライアント名**: テキスト入力
2. **プロジェクト名**: テキスト入力
3. **作成者名**: テキスト入力（見積書の作成者として記載）
4. **単価（人日）**: 選択肢で提示

| 選択肢 | 用途 |
|--------|------|
| ¥50,000 /人日 | ジュニアエンジニア |
| ¥70,000 /人日 | ミドルエンジニア |
| ¥100,000 /人日 | シニアエンジニア |
| カスタム金額を入力 | 上記以外の単価 |

4. **有効期限**: デフォルトは見積日より30日間。変更が必要か確認
5. **フェーズ分割**: フェーズに分けるか確認

| 選択肢 | 説明 |
|--------|------|
| フェーズ分割なし | 全機能を1期で見積もり |
| フェーズ分割あり | カテゴリや優先度でフェーズに分割 |

### Step 3: 金額の計算

以下のルールで計算する:

| 項目 | 計算方法 |
|------|---------|
| 各機能の金額 | 工数（人日） × 単価 |
| 小計（税抜） | 全機能の金額の合計 |
| 消費税 | 小計 × 10% |
| 合計（税込） | 小計 + 消費税 |

端数処理:
- 各機能の金額: 1円未満切り捨て
- 消費税: 1円未満切り捨て

### Step 4: 見積書の生成

`knowledge/templates/estimate-doc.md` のテンプレートに従って見積書を生成する。

#### フェーズ分割ありの場合

カテゴリまたは優先度に基づいてフェーズを自動分割する:

1. カテゴリごとにグルーピング
2. 各フェーズの工数・金額を集計
3. フェーズ別スケジュールセクションを追加（1人日 = 1営業日として期間を概算）

#### 前提条件の自動生成

以下の前提条件を自動的に含める:

- 本見積りは提示された要件に基づくものであり、要件変更時は再見積りとする
- テスト工数は単体テストを含む。結合テスト・システムテストが必要な場合は別途見積り
- インフラ・サーバー費用は含まない（必要に応じて別途見積り）
- デザイン制作費は含まない（別途見積り、または支給の場合）

#### 含まれないものの自動生成

スプレッドシートの備考欄に「スコープ外」「別途」等の記載がある項目を自動的に抽出する。

### Step 5: 出力ファイルの保存

出力先: `docs/outputs/{案件slug}/estimate-doc.md`
- `{案件slug}` はプロジェクト名から英語の短いキーワードで生成（英数字・ハイフン）。同じ案件の別コマンド出力と同じ slug を使う
- ディレクトリが存在しない場合は作成する。既に同じ案件フォルダがあればその中に追加する

### Step 6: 結果の報告

以下を報告する:

- **出力ファイル**: 見積書のパス
- **見積概要**: 総工数、単価、合計金額（税込）
- **機能数**: 見積対象の機能数
- **フェーズ数**: フェーズ分割した場合のフェーズ数

推奨する次のアクション:

| アクション | コマンド |
|-----------|---------|
| 見積書のレビュー | `/reviewing docs/outputs/{案件slug}/estimate-doc.md` |
| PR 作成 | `/creating-pr 見積書を作成` |

## 注意事項

- 単価はクライアント・案件によって異なるため、必ずユーザーに確認する
- 工数データがスプレッドシートにない（空欄）機能がある場合は警告し、先に `/estimating` の実行を提案する
- 金額の端数処理は一貫して切り捨てを使用する
- 見積書はあくまで概算であり、最終的な金額はユーザーの判断による旨を備考に記載する

## 関連コマンド

- `/estimating` — 工数見積もりを行う（この入力となるスプレッドシートを生成）
- `/define-requirements` — 要件定義書を生成する（パイプラインの上流）
- `/reviewing` — 見積書のレビューを行う
- `/creating-pr` — 見積書を PR として提出する
