# 成果物レビューチェックリスト

実行結果（`docs/outputs/` 配下）をレビューする際に確認する観点です。

## 必須チェック項目

### 仕様との整合性
- [ ] 仕様書の要件をすべて満たしているか
- [ ] 仕様書にない余計なものが含まれていないか
- [ ] 仕様書の想定成果物と一致しているか

### 品質
- [ ] テンプレート（`knowledge/templates/`）に沿っているか
- [ ] 誤字脱字・フォーマットの乱れがないか
- [ ] 内容に矛盾や論理的な飛躍がないか
- [ ] 数値やデータに明らかな誤りがないか

### 安全性
- [ ] 機密情報（API キー・パスワード・個人情報）が含まれていないか
- [ ] 社外に出して問題ない内容か
- [ ] ファイルの保存先が正しいか（`docs/outputs/` 配下か）

### 完了状態
- [ ] タスクリストのチェックがすべて完了しているか
- [ ] 実行レポートが生成されているか
- [ ] 未解決の問題・次のアクションが明記されているか

## 判定基準

| 判定 | 意味 | 次のアクション |
|------|------|--------------|
| ✅ 承認 | すべての必須項目をクリア | PR 作成に進んでOK |
| ⚠️ 条件付き承認 | 軽微な修正が必要 | 指摘箇所を修正してから PR |
| ❌ 差し戻し | 重大な問題あり | 再実行または仕様書から見直し |
