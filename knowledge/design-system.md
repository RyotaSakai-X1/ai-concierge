# デザインシステム

## 概要

このドキュメントはプロジェクトのデザイントークン選定における**基準・プロセス・出力粒度**を定義する。
具体的な色値・フォント・スケールは案件ごとにトレンド調査と要件に基づいて都度決定する。

> 📌 本ドキュメントで定義しているトークン体系・コンポーネント構成・UIパターン自体が、将来のトレンドで陳腐化する可能性がある。案件開始時のトレンド調査で、ここに定義された前提（ヘッダーの存在、サイドバー型ナビゲーション、8pxグリッド等）が現在の主流と乖離していると判断した場合は、本ドキュメントの定義に縛られずトレンドを優先し、その結果をフィードバックとして本ドキュメントに反映すること。

## カラーパレット

### セマンティックカラー体系

案件ごとに以下のトークンを全て定義する。色値は調査結果から導出し、この体系に当てはめる。

| トークン名 | 用途 | 選定時の注意 |
|-----------|------|------------|
| primary | ブランドカラー、主要アクション（CTA ボタン、リンク） | 白背景でコントラスト比 4.5:1 以上必須 |
| primary-hover | primary のホバー状態 | primary より 1-2 段階暗い色。コントラスト比は primary 以上になる |
| primary-light | primary の薄い背景色（バッジ背景、選択状態） | primary の色相を保ちつつ彩度・明度を大幅に上げる |
| secondary | 補助的なアクション、セカンダリボタン | primary との差別化が明確であること。ニュートラル系が多い |
| accent | 強調・ハイライト、注目させたい要素 | primary と色相が離れていること。テキストには使わず装飾・アイコン用途 |
| neutral-50 ~ 900 | 背景・ボーダー・テキスト（段階的に暗くなる） | 50/100 は背景、200 はボーダー、600 以上はテキスト |
| success | 成功状態 | 緑系。テキスト使用時は必ずアイコンを併用 |
| warning | 警告状態 | 橙～黄系。コントラスト比が低くなりがちなので 4.5:1 以上を確保するか、アイコン必須にする |
| error | エラー状態 | 赤系。白背景で 4.5:1 以上必須 |
| info | 情報状態 | 青系。primary と同色でもよい |

### カラー選定基準

案件のカラーパレットを決める際、以下の基準を全て満たすこと。

**アクセシビリティ（必須）:**
- テキスト色は白背景で WCAG AA 準拠（コントラスト比 4.5:1 以上）
- 大テキスト（14pt bold ≒ 18.7px bold / 18pt regular = 24px regular 以上）は 3.0:1 以上
- プレースホルダー・無効状態は WCAG 対象外だが、2.0:1 以上を推奨
- success / warning をテキストとして使う場合は必ずアイコンを併用するか、コントラスト比 4.5:1 以上の色を選ぶ
- 色だけで状態を区別しない（アイコン・テキストを併用）

**色彩理論:**
- primary と accent の色相差は 60° 以上（補色・分裂補色・トライアドなど）
- neutral は pure gray または微量のブランドカラーを含む（Slate, Gray, Zinc 等）
- semantic colors（success/warning/error）はトーンに関係なく、認知的に自然な色相を使う（緑/橙/赤）

**実用性:**
- Tailwind CSS のカラーパレットをベースにすると開発効率が高い（Slate, Gray, Zinc, Blue, Violet 等の 50-900 スケール）
- ダークモード対応が必要な場合は別途設計ガイドを作成する（surface / on-surface カラーの追加定義が必要）

### カラー選定プロセス

1. **トレンド調査** — 案件の業界・ターゲットに合ったカラートレンドを Web 検索で調査
2. **候補生成** — 調査結果と要件から 2-3 案のパレットを生成。各候補に以下を含める:
   - トーンの方向性（キーワード 3-5 個）
   - 全トークンの色値（HEX）
   - 各色の白背景コントラスト比と WCAG AA 合否
   - 選定根拠（なぜこの色相・彩度・明度にしたか）
3. **ユーザー選定** — 候補をユーザーに提示し、選択してもらう
4. **確定** — 選択されたパレットを案件の成果物フォルダに記録

## タイポグラフィ

### フォント選定基準

| 基準 | 内容 |
|------|------|
| 日本語対応 | 日本語ウェイトが 4 段階以上あること |
| Variable Font | 対応していればファイルサイズと表現力の両立が可能 |
| UI 最適化 | 数字の等幅（tabular figures）、小文字の視認性が高い |
| ライセンス | Google Fonts またはオープンソースを推奨 |

### フォントファミリー（デフォルト推奨）

案件固有の要件がなければ以下を採用する。要件に応じて変更可。

| 用途 | フォント | フォールバック | 理由 |
|------|--------|-------------|------|
| 日本語 | Noto Sans JP | sans-serif | Google Fonts 無料。ウェイト 7 段階で表現力が高い。可読性が高く汎用的 |
| 欧文 | Inter | system-ui, sans-serif | Variable Font 対応。字間・数字のデザインが UI 向けに最適化されている |
| 等幅 | JetBrains Mono | monospace | コードブロック・技術情報用。リガチャ対応 |

### モジュラースケール

**ベースサイズ**: 16px（1rem）
**スケール比**: 1.250（Major Third）

| トークン | サイズ | 行間 | 用途 |
|---------|-------|------|------|
| text-xs | 12px (0.75rem) | 1.5 (18px) | キャプション、バッジ、ヘルパーテキスト |
| text-sm | 14px (0.875rem) | 1.5 (21px) | テーブル、フォームラベル、サイドバー |
| text-base | 16px (1rem) | 1.6 (25.6px) | 本文、段落 |
| text-lg | 20px (1.25rem) | 1.5 (30px) | セクション見出し（h3） |
| text-xl | 約25px (1.563rem) | 1.3 (32.5px) | ページ内見出し（h2） |
| text-2xl | 約31px (1.953rem) | 1.2 (37.2px) | ページタイトル（h1） |
| text-3xl | 約39px (2.441rem) | 1.1 (42.9px) | ヒーローセクション |

> ⚠️ 仮定：モジュラースケール比 1.250 は管理画面・コーポレートサイト両方で使いやすいバランス。LP で見出しをもっと大きくしたい場合は 1.333（Perfect Fourth）への変更も検討可。

> 📌 text-xs / text-sm は Tailwind CSS 準拠の実用値を採用。厳密なモジュラースケール（16/1.25=12.8px, 12.8/1.25=10.24px）ではなく、可読性を優先した値。text-lg 以上はスケール比 1.250 に準拠。

## コンポーネントカタログ

### 設計方針

Atomic Design の階層構造（Atoms → Molecules → Organisms → Templates → Pages）を基本とする。本ドキュメントでは Atoms と Molecules の仕様を定義する。Organisms 以降は案件固有のレイアウトに依存するため、案件ごとに設計する。

> 📌 Atomic Design は現時点で広く採用されている設計手法だが、将来的にコンポーネント分類の主流が変わる可能性がある。案件開始時のトレンド調査で、Atomic Design の階層構造が案件の要件やトレンドに合わないと判断した場合は、代替手法（Feature-based, Domain-driven 等）を採用し、その結果をフィードバックとして本ドキュメントに反映すること。

### ブレない軸（全コンポーネント共通）

以下の基準はトレンドに関係なく遵守する。

**命名規則:**

| ルール | 内容 |
|--------|------|
| ケース | PascalCase（例: `PrimaryButton`, `TextInput`） |
| 接頭辞 | 階層を示す接頭辞は付けない。ディレクトリ構造で分類する |
| バリアント名 | size は `sm` / `md` / `lg`、variant は意味ベース（`primary` / `secondary` / `danger` 等） |
| 状態名 | HTML 標準に準拠（`disabled`, `readonly`, `required`, `checked` 等） |

**アクセシビリティ（必須）:**

| 基準 | 内容 |
|------|------|
| キーボード操作 | フォーカス可能な要素は Tab / Enter / Space / Escape で操作できること |
| ARIA | ネイティブ HTML 要素で表現できない場合のみ `aria-*` 属性を使用する |
| コントラスト | テキスト要素は WCAG AA 準拠（カラーパレットセクションの基準に従う） |
| フォーカスリング | `:focus-visible` で視認可能なフォーカスインジケータを表示する |

**状態管理:**

全インタラクティブコンポーネントは以下の状態を定義する。

| 状態 | 説明 |
|------|------|
| default | 初期状態 |
| hover | マウスオーバー時 |
| active | クリック・タップ中 |
| focus | キーボードフォーカス時 |
| disabled | 操作不可 |
| loading | 処理中（該当する場合） |

### コンポーネント選定プロセス

案件開始時に以下のプロセスでコンポーネント構成を決定する。

1. **トレンド調査** — 案件の業界・ターゲットに合った UI コンポーネントのトレンドを調査
2. **要件照合** — 画面設計・ワイヤーフレームから必要なコンポーネントを洗い出す
3. **カタログ照合** — 本カタログの Atoms / Molecules で充足するか確認する
4. **差分対応** — 不足があれば追加定義、不要なものはスキップ。トレンドとの乖離があれば本カタログを更新する

### Atoms

UI の最小構成単位。単体で意味を持ち、他のコンポーネントに依存しない。

#### Button

| 項目 | 定義 |
|------|------|
| 用途 | ユーザーアクションのトリガー |
| バリアント | `primary` / `secondary` / `outline` / `ghost` / `danger` |
| サイズ | `sm` / `md` / `lg` |
| 状態 | default / hover / active / focus / disabled / loading |
| アイコン対応 | 左アイコン / 右アイコン / アイコンのみ（icon-only） |
| ガイドライン | CTA には `primary` を使用。1画面に `primary` は原則1つ。破壊的操作には `danger` を使用 |

#### Input

| 項目 | 定義 |
|------|------|
| 用途 | テキスト入力 |
| バリアント | `text` / `password` / `email` / `number` / `search` / `tel` / `url` |
| サイズ | `sm` / `md` / `lg` |
| 状態 | default / hover / focus / disabled / readonly / error |
| 付随要素 | ラベル（必須） / プレースホルダー / ヘルパーテキスト / エラーメッセージ / 接頭辞・接尾辞アイコン |
| ガイドライン | ラベルは省略しない（アクセシビリティ）。エラー時は色だけでなくメッセージテキストで伝える |

#### Select

| 項目 | 定義 |
|------|------|
| 用途 | 選択肢からの単一選択 |
| サイズ | `sm` / `md` / `lg` |
| 状態 | default / hover / focus / disabled / error |
| 付随要素 | ラベル（必須） / プレースホルダー / ヘルパーテキスト / エラーメッセージ |
| ガイドライン | 選択肢が 5 件以下ならラジオボタンも検討する。ネイティブ `<select>` を基本とし、カスタム UI が必要な場合のみ Dropdown（Molecule）を使用 |

#### Checkbox

| 項目 | 定義 |
|------|------|
| 用途 | 複数選択、同意確認 |
| サイズ | `sm` / `md` |
| 状態 | default / hover / focus / disabled / checked / indeterminate |
| 付随要素 | ラベル（必須） |
| ガイドライン | ラベルクリックでも切り替え可能にする。グループ時は Fieldset + Legend でまとめる |

#### Radio

| 項目 | 定義 |
|------|------|
| 用途 | 排他的な単一選択 |
| サイズ | `sm` / `md` |
| 状態 | default / hover / focus / disabled / checked |
| 付随要素 | ラベル（必須） |
| ガイドライン | 選択肢が 6 件以上なら Select を検討する。Fieldset + Legend でグループ化する |

#### Badge

| 項目 | 定義 |
|------|------|
| 用途 | ステータス表示、カウント、ラベリング |
| バリアント | `default` / `success` / `warning` / `error` / `info` |
| サイズ | `sm` / `md` |
| ガイドライン | セマンティックカラーと対応させる。ドット型（テキストなし）は通知カウント用途に限定 |

#### Icon

| 項目 | 定義 |
|------|------|
| 用途 | 視覚的な補助、アクション示唆 |
| サイズ | `sm`(16px) / `md`(20px) / `lg`(24px) |
| 提供方式 | SVG インライン（色・サイズ変更に対応） |
| ガイドライン | 装飾目的のアイコンは `aria-hidden="true"` を設定する。意味を持つアイコンには `aria-label` を付与する |

#### Label

| 項目 | 定義 |
|------|------|
| 用途 | フォーム要素の説明 |
| 付随表示 | 必須マーク（`*` または「必須」テキスト） |
| ガイドライン | `<label>` 要素と `for` 属性で入力要素に紐づける。必須項目は視覚的にもスクリーンリーダー的にも明示する |

#### Textarea

| 項目 | 定義 |
|------|------|
| 用途 | 複数行テキスト入力 |
| サイズ | `sm` / `md` / `lg`（高さの初期値） |
| 状態 | default / hover / focus / disabled / readonly / error |
| 付随要素 | ラベル（必須） / プレースホルダー / ヘルパーテキスト / エラーメッセージ / 文字数カウンター |
| ガイドライン | リサイズは垂直方向のみ許可を推奨。最大文字数がある場合はカウンターを表示する |

#### Toggle（Switch）

| 項目 | 定義 |
|------|------|
| 用途 | ON/OFF の即時切り替え |
| サイズ | `sm` / `md` |
| 状態 | default / hover / focus / disabled / checked |
| ガイドライン | 保存操作を必要とする場合は Checkbox を使用する。Toggle は即時反映される設定に限定 |

### Molecules

複数の Atoms を組み合わせた複合コンポーネント。特定の機能を実現する。

#### Card

| 項目 | 定義 |
|------|------|
| 用途 | 関連情報のグルーピング表示 |
| 構成 Atoms | 見出しテキスト / 本文テキスト / Badge / Button / Icon |
| バリアント | `default` / `clickable`（全体がリンク） / `with-image` |
| 構成要素 | ヘッダー（任意） / ボディ（必須） / フッター（任意） / メディア（任意） |
| ガイドライン | `clickable` バリアントは `<a>` または `role="link"` で実装する。カード内に複数のアクションがある場合は `clickable` にしない |

#### Modal（Dialog）

| 項目 | 定義 |
|------|------|
| 用途 | 確認ダイアログ、フォーム入力、詳細表示 |
| 構成 Atoms | 見出しテキスト / Button / Icon（閉じるボタン） |
| バリアント | `default` / `confirm`（確認ダイアログ） / `fullscreen` |
| サイズ | `sm` / `md` / `lg` |
| 構成要素 | ヘッダー（タイトル + 閉じるボタン） / ボディ / フッター（アクションボタン群） |
| ガイドライン | 背景オーバーレイで他要素を操作不可にする。Escape キーで閉じる。フォーカストラップを実装する。`<dialog>` 要素の使用を推奨 |

#### FormField

| 項目 | 定義 |
|------|------|
| 用途 | ラベル + 入力要素 + バリデーションを1つにまとめた単位 |
| 構成 Atoms | Label / Input（または Select, Textarea, Checkbox, Radio） / ヘルパーテキスト / エラーメッセージ |
| 状態 | default / error / success |
| ガイドライン | バリデーションエラーは入力フィールド直下に表示する。リアルタイムバリデーションはフォーカスアウト時に実行する |

#### Alert

| 項目 | 定義 |
|------|------|
| 用途 | システムメッセージ、操作結果の通知 |
| 構成 Atoms | Icon / テキスト / Button（閉じる、任意） |
| バリアント | `success` / `warning` / `error` / `info` |
| 構成要素 | アイコン / タイトル（任意） / メッセージ / アクション（任意） |
| ガイドライン | セマンティックカラーの Badge と同じ色体系を使用する。`role="alert"` または `role="status"` を設定してスクリーンリーダーに通知する |

#### Dropdown

| 項目 | 定義 |
|------|------|
| 用途 | カスタムスタイルの選択肢表示、アクションメニュー |
| 構成 Atoms | Button（トリガー） / Icon / テキスト |
| バリアント | `select`（選択用途） / `menu`（アクション用途） |
| 構成要素 | トリガー / ドロップダウンパネル / 選択肢リスト / セパレーター（任意） |
| ガイドライン | 上下矢印キーで選択肢を移動、Enter で確定、Escape で閉じる。`select` バリアントはネイティブ `<select>` で不十分な場合のみ使用する。`aria-expanded`, `aria-haspopup` を設定する |

#### Table

| 項目 | 定義 |
|------|------|
| 用途 | 構造化データの一覧表示 |
| 構成 Atoms | テキスト / Badge / Button / Checkbox（行選択） / Icon |
| 機能 | ソート（任意） / ページネーション（任意） / 行選択（任意） |
| 構成要素 | ヘッダー行 / データ行 / フッター（任意） |
| ガイドライン | `<table>` / `<thead>` / `<tbody>` / `<th scope>` を正しく使用する。レスポンシブ対応は横スクロールまたはカード型への切り替えで対応する |

#### Tabs

| 項目 | 定義 |
|------|------|
| 用途 | コンテンツの切り替え表示 |
| 構成 Atoms | Button（タブ） / Badge（カウント表示、任意） / Icon（任意） |
| バリアント | `default` / `underline` / `pill` |
| ガイドライン | `role="tablist"` / `role="tab"` / `role="tabpanel"` を設定する。左右矢印キーでタブ間を移動する |

#### Pagination

| 項目 | 定義 |
|------|------|
| 用途 | 複数ページのナビゲーション |
| 構成 Atoms | Button / Icon（矢印） / テキスト |
| 構成要素 | 前へ / ページ番号 / 省略記号 / 次へ |
| ガイドライン | `nav` 要素と `aria-label="ページナビゲーション"` で囲む。現在ページは `aria-current="page"` を設定する |

### コンポーネント仕様テンプレート

案件固有のコンポーネントを追加定義する際は、以下のテンプレートに従う。

| 項目 | 記載内容 |
|------|---------|
| 用途 | このコンポーネントが何のために存在するか |
| 構成 Atoms | （Molecule の場合）使用する Atoms の一覧 |
| バリアント | 見た目・用途の違いによるバリエーション |
| サイズ | サイズバリエーション（該当する場合） |
| 状態 | インタラクション状態の一覧 |
| 構成要素 | 内部スロット・領域の定義 |
| アクセシビリティ | ARIA 属性、キーボード操作の要件 |
| ガイドライン | 使い分けの基準、アンチパターン |
